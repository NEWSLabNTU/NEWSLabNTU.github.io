215/* ---- Threading: wake primitive (Phase 129) ----
216 *
217 * Binary-semaphore-shaped primitive used by the executor's wake_flag /
218 * spin_once cv-wait pair. Separate from `nros_platform_condvar_*` so
219 * the executor doesn't inherit zenoh-pico's pthread-shaped Zephyr
220 * ABI (which on libc hangs past `pthread_cond_timedwait` deadlines).
221 *
222 * Per-platform impl:
223 * * POSIX: `sem_t` with `sem_timedwait` (`CLOCK_MONOTONIC`).
224 * * Zephyr: `k_sem` (kernel-native, libc-pthread-free).
225 * * FreeRTOS: `xSemaphoreCreateBinary` + `xSemaphoreGiveFromISR`.
226 * * NuttX: POSIX `sem_t` via NuttX libc (`sem_timedwait`).
227 * * ThreadX: `tx_semaphore` (ISR-safe `tx_semaphore_put`).
228 * * bare: atomic flag + busy-spin against the platform clock.
229 *
230 * `wait_ms` returns 0 on signal, 1 on timeout, -1 on error.
231 * `signal_from_isr` returns -1 if the backend has no ISR-safe path;
232 * callers may fall back to `signal()` accepting the latency cost.
233 * Storage is opaque to the caller; size/alignment via the probe
234 * helpers below. */

246/* ---- Critical section ---- */
247/* Phase 121.9 — global mutual exclusion against preemption + ISR
248 * delivery. Backs the Rust `critical_section::Impl` registration used
249 * by dust-dds, nros-rmw-{xrce,zenoh}, and any consumer of
250 * `critical_section::with()`. Reentrant by contract: every acquire
251 * must be paired with exactly one release, and the platform handles
252 * nesting (PRIMASK already stacks; pthread side uses a recursive
253 * mutex).
254 *
255 * The `uint32_t` token holds whatever the platform needs to restore
256 * the prior posture — Cortex-M PRIMASK bit, Cortex-R CPSR I-bit,
257 * RISC-V `mstatus.MIE` snapshot, pthread depth counter — and is
258 * opaque to the caller. */
