Phase 219.C — C Entry-pkg TU emitter.

Maps a Plan (see super::mod) onto a generated main.c body per docs/roadmap/archived/phase-219-cpp-entry-pkg.md §3.3 + §3.5. Symmetric to super::emit_cpp modulo C ABI: register fns return nros_ret_t (not int32_t) and take nros_node_context_t* (not ::nros::NodeContext*), and the Board adapter is the C free-fn nros_board_native_run instead of the templated C++ ::nros::board::NativeBoard::run.

The Phase 219.E sibling header <nros/main.h> exports nros_board_native_run(nros_node_register_fn entry) (one fn-ptr, no closure); the generated main() packs the per-Node register sequence into a static fn that the adapter receives.
