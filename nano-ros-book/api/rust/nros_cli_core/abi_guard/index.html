Phase 218.E — ABI version guard between the prebuilt nros CLI and the nros-core runtime crate the consumer’s Cargo.lock resolves.

The CLI emits Rust / C / C++ code that targets nros-core, nros-c, and nros-cpp runtime ABIs. A version mismatch between the CLI binary and the runtime crates the consumer links surfaces as link errors, struct-layout mismatches, or — worst — silent runtime UB. The guard catches this BEFORE codegen runs.

§Surface

  • check_workspace — resolves the authoritative Cargo.lock, parses it, finds the resolved nros-core version, compares to the CLI binary’s embedded version (CLI_VERSION), and either continues or exits with an actionable error message naming both versions plus the fix command. For consumers inside the nano-ros monorepo the lock is the monorepo root lock (in-tree examples link the patched in-tree nros-core, so a standalone crate’s own — possibly stale — lock is not authoritative); external consumers use the nearest Cargo.lock.
  • check_workspaces — the multi-workspace variant for verbs that resolve inputs from more than one workspace (e.g. nros codegen-system).

§Opt-out

NROS_SKIP_VERSION_CHECK=1 in the environment bypasses the check, with a warning: line on stderr so the bypass is visible in CI logs.

§Match rule

Strict equality on the full SemVer string today. The comparison fn (versions_match) is the single point to relax later (e.g. to “MAJOR.MINOR equal”).

§Missing Cargo.lock

If the consumer has not run cargo generate-lockfile yet, the guard warns-and-continues — the codegen step the verb is about to run will probably create the lock itself.

The codegen verb that triggered the check — flows into the error message so the user knows which command tripped the guard.

Constants§

CLI_VERSION
The CLI binary’s embedded version — baked at compile time from the nros-cli-core crate’s CARGO_PKG_VERSION.
SKIP_ENV
Env-var name that bypasses the guard (any non-empty value opts out).

Functions§

check_workspace
Run the guard against a single workspace anchor.
check_workspaces
Multi-workspace variant — checks each anchor in turn and fails on the first mismatch. Used by verbs that resolve inputs from more than one workspace (e.g. nros codegen-system reading a bringup pkg from workspace A + member pkgs from workspace B).
find_cargo_lock
Walk up from start looking for a Cargo.lock. Returns None if none is found above the filesystem root.
find_monorepo_root
Walk up from start to find the nano-ros monorepo root — the directory containing packages/core/nros-core/Cargo.toml. Returns None when start is not inside the monorepo (a genuine external consumer), in which case the nearest-Cargo.lock rule applies.
nros_core_version_in_lock
Parse a Cargo.lock (TOML) and pull out the resolved version of the named package, if present.
versions_match
Comparison rule. Strict equality today. The single point to relax later (e.g. compare only MAJOR.MINOR).
