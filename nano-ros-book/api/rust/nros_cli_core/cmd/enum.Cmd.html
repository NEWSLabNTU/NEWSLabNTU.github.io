§

Codegen(Args)

Build-tool C/C++ binding generation (--args-file / resolve-deps). The interface the cmake / build.rs consumers speak (Phase 195.A — folds in the former standalone nros-codegen binary).

§

CodegenSystem(Args)

Host-time system bake (Phase 212.E) — read <bringup>/system.toml + <bringup>/launch/system.launch.xml and emit the baked compile-time C config + component registration glue consumed by every embedded RTOS adapter.

§

Metadata(Args)

Collect component source metadata for orchestration planning

§

Migrate(MigrateSub)

Migrate a pre-212 workspace to the new shape (Phase 212.I).

Hidden from nros --help: this is an internal maintainer tool that runs once per pre-212 workspace and retires. End users start from the post-212 shape (nros new system <bringup>) and never touch this verb. Kept callable via cargo run -p nros-cli for the in-tree fixture sweep.

§

Plan(Args)

Resolve launch files, manifests, and metadata into nros-plan.json

§

Check(Args)

Validate a generated nros-plan.json

§

Explain(Args)

Render a generated nros-plan.json in human-readable form

§

Config(Args)

Inspect or validate the current project’s resolved configuration

§

Build(Args)

§

Deploy(Args)

Run a [deploy.] target from the root nros.toml (Phase 172 WP-A)

§

Launch(Args)

Spawn a bringup pkg’s components on the host (Phase 212.J — no ament install required; the desktop / native_sim alternative to ros2 launch).

§

Setup(Args)

Resolve + fetch a board’s toolchain/SDK packages (Phase 187)

§

Run(Args)

Build, flash, and monitor the current project on the selected target
