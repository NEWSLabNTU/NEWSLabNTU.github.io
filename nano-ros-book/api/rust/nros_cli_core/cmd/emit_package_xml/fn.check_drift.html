pub fn check_drift(pkg_dir: &Path) -> Result<DriftStatus>
Expand description

Phase 212.G.2 drift check. Re-renders package.xml for the given pkg dir and compares it against the existing file (if present + generator-owned).

Returns:

  • Ok(None) — file absent, or carries no generated marker (hand-written package.xml — out of scope for drift; user owns it).
  • Ok(Some(())) — file matches a fresh emit. No drift.
  • Err(_) — file carries the marker but DIFFERS from a fresh emit: the diagnostic body describes the drift. Caller (nros check) prints it as a warning.
