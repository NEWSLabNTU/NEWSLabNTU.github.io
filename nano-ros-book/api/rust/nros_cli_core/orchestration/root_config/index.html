Phase 172 WP-A — root nros.toml workspace config (the deployment SSOT).

One config file at the workspace root holds everything for the revised deployment model: [workspace] defaults, the [system] / [systems.<name>] composition (launch + components + the RMW/domain SSOT + overlays + scheduling + in-binary [[domain]]/[[bridge]]), and the [deploy.<name>] targets the nros deploy command-runner drives. This replaces the per-package “system nros.toml with target.{triple,board}” — triple/board move into [deploy.<name>]. Component nros.toml stays a different scope (reusable intrinsics) and must not carry rmw/domain.
