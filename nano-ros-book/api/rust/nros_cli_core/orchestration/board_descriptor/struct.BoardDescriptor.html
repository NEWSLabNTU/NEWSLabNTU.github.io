Expand description

One board profile. Mirrors the old hardcoded PlatformProfile + BoardEntry, but every field is owned data read from nros-board.toml.

Fields§

§names: Vec<String>

Board name + accepted aliases (the values a user passes as board).

§platform: PlatformKind§target: Option<String>

rustc target triple this board pins, if any (None → take from plan).

§toolchain: Toolchain§platform_feature: String

The nros/<feature> selected (e.g. platform-posix).

§local_aliases: Vec<String>

Extra local default-feature aliases beyond nros/<feature>.

§link_kind: LinkKind§entry_kind: EntryKind§net_stack: NetStack§chip: Option<String>

esp-hal / stm32 chip feature; None for non-chip platforms.

§board_crate: Option<String>

Board crate to depend on; None for crate-less host boards (posix / zephyr / orin-spe) that pull static or nros-platform-cffi deps.

§crate_path: Option<String>

Board-crate path relative to the workspace root; defaults to packages/boards/<board_crate> when omitted.

§board_features: Vec<String>

Extra features to enable on the board crate dependency.

§cargo_config: Option<String>

Verbatim .cargo/config.toml body, with ${workspace} placeholders for any layout path. None for boards that need no config (posix/zephyr/…).

§entry: Option<BoardEntry>

Generated entry-point pieces; None for hosted boards that emit the default fn main shape.

§target_contains: Option<String>

Disambiguate two descriptors sharing a names entry by requiring this substring in the requested target (e.g. "riscv64" for threadx-riscv64, so board = "threadx" picks riscv64 vs linux by target).
