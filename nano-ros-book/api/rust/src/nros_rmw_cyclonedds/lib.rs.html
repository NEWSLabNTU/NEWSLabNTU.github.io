1//! Phase 212.K.7 — Rust shim for the Cyclone DDS RMW backend.
2//!
3//! Co-located with the C++ implementation under
4//! `packages/dds/nros-rmw-cyclonedds/`: the C++ TUs (`src/*.cpp`) are
5//! built by the sibling `CMakeLists.txt`; this crate adds the Rust
6//! side that replaces the per-msg-pkg `cyclonedds` Cargo feature
7//! pattern. See `docs/roadmap/phase-212-…-file-consolidation.md`
8//! §212.K.7 for the design.
9//!
10//! # Surface
11//!
12//! * [`dynamic_type::DescriptorBuilder`] — walks
13//!   [`nros_serdes::schema::Message`]'s static field schema and asks
14//!   the C++ bridge to construct a Cyclone `dds_topic_descriptor_t`
15//!   at runtime via Cyclone's dynamic-type API.
16//! * [`type_registry::TypeRegistry`] — bounded
17//!   [`heapless::FnvIndexMap`] of type-name → descriptor pointer. The
18//!   global [`type_registry::global`] is mutex-guarded; first
19//!   [`type_registry::TypeRegistry::get_or_build`] for a given `M`
20//!   builds via `DescriptorBuilder` and caches the result.
21//!
22//! # Contract
23//!
24//! Hard constraints (enforced by CI + acceptance):
25//!
26//! * `#![no_std]`. No `alloc`, no `std`. The `std` feature is a
27//!   reserved escape hatch for future hosted-only diagnostics; nothing
28//!   in this crate touches it today.
29//! * Storage is fixed-capacity [`heapless`]; overflow returns
30//!   [`dynamic_type::BuildError::RegistryFull`] (no panic, no heap).
31//! * Synchronisation: [`spin::Mutex`] on hosted POSIX/macOS,
32//!   [`critical_section::Mutex`] on `target_os = "none"`. See
33//!   [`sync`].
34//! * The C++ bridge allocates descriptors from Cyclone's `ddsrt`
35//!   heap (Phase 177.22's pre-budgeted pool on embedded targets), not
36//!   from libc malloc.
37//!
38//! # Integration status
39//!
40//! K.7.6 ("wire pub/sub paths") is partially landed: the registry
41//! exposes [`type_registry::register`] as the public entry, but the
42//! existing C++ pub/sub paths
43//! (`packages/dds/nros-rmw-cyclonedds/src/publisher.cpp`,
44//! `subscriber.cpp`, `service.cpp`) still resolve descriptors through
45//! the static-init [`descriptors.cpp`](../descriptors.cpp) table.
46//! Wiring the Rust registry into the Rust-side `nros-node`
47//! `create_publisher` / `create_subscription` paths is the K.7.7
48//! / K.7.6 follow-up that lands once `nros-cli`'s codegen
49//! template emits `impl Message for …` on every generated msg
50//! crate (K.7.1). Until then, the registry is exercised by the
51//! unit tests in `tests/`.
52//!
53//! # `#[cfg(test)]` shim
54//!
55//! Tests build under `cargo test -p nros-rmw-cyclonedds` on a hosted
56//! toolchain that has no Cyclone DDS C library available. The
57//! [`bridge`] module exposes a `#[cfg(test)]` stub for
58//! `nros_cyclonedds_build_descriptor_from_schema` so the registry +
59//! builder logic can be tested in isolation. Production builds link
60//! the real symbol from the C++ TU added in
61//! `bridge/dynamic_type_builder.cpp`.
62
