Expand description

BSP shim — register every component in register_fns against runtime, then spin until halt. The Phase 212.M.5.a.3 baker’s cmake-emitted system_main.c collects the per-pkg register fn pointers and calls into a single Rust shim, which holds the C ABI mismatch on the Rust side and never exposes ComponentContext across the FFI boundary.

Each register fn is invoked under a fresh ComponentContext backed by a private one-shot ComponentRuntime sink — that gives every pkg its own declaration-time view of the runtime without the BSP needing to know the per-pkg Component type. The callback bodies are NOT routed from this path today (see [DeclarativeSlot] and the M.5.a.3 follow-up); call sites that need full callback dispatch use ExecutorComponentRuntime::register_component directly with the concrete component type.
