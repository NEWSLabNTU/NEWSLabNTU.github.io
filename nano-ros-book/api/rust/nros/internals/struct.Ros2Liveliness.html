ROS 2 liveliness key expression builder for the shim transport

Generates the key expressions required for ROS 2 discovery via rmw_zenoh.

Protocol format: {PREFIX}/{domain_id}/{zid}/{version}/{entity}/%/{ns}/{node}[/{topic}/{type}/{hash}/{qos}]

pub fn node_keyexpr<const N: usize>( domain_id: u32, zid: &ZenohId, namespace: &str, node_name: &str, ) -> String<N>

Build a node liveliness key expression

Format: @ros2_lv/<domain_id>/<zid>/0/0/NN/%/<mangled_ns>/<node_name>

The namespace is mangled using the same rules as topic names:

  • / → %
  • /demo → %demo
  • /ns/sub → %ns%sub

pub fn service_server_keyexpr_wildcard<const N: usize>( domain_id: u32, service: &ServiceInfo<'_>, ) -> String<N>

Build a wildcard liveliness keyexpr matching any service-server token for the given service.

Format: @ros2_lv/<domain_id>/*/0/11/SS/%/*/*/<service>/<type>/*/* — wildcards on zid, namespace, node, type_hash, and qos. Used by Client::wait_for_service to discover any matching server before the first request, mirroring rclcpp::ClientBase::wait_for_service.
