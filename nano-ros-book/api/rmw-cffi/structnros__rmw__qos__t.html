Detailed Description

Minimal QoS subset honoured by every nros RMW backend.

The full DDS QoS profile family (deadline, lifespan, liveliness, partition, ownership, …) is not represented here — backends honour the subset they natively implement, no more. See the book concepts/ros2-comparison.md "QoS subset, not full DDS profiles" section for the rationale.

depth is uint16_t (max 65 535). Embedded ROS application queue depths are typically 1–100; the 16-bit width saves two bytes per entity vs the upstream 32-bit choice.
